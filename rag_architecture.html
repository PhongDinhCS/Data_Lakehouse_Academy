Production RAG Architecture

1. Big Picture

Offline Pipeline (Build Knowledge)
+
Online Pipeline (Serve User)

2. Full System Flow

DOCUMENT PIPELINE
Documents → Parsing → Chunk → Embedding → Vector DB

QUERY PIPELINE
User Query → Embedding → Retrieval → Ranking → LLM → Answer

3. Architecture Layers

1. Data Ingestion Layer
2. Processing Layer (Parsing + Chunking)
3. Embedding Layer
4. Storage Layer (Vector DB + Metadata DB)
5. Retrieval Layer
6. LLM Layer
7. Feedback & Monitoring Layer

4. Offline Pipeline (Build Knowledge)

Documents
PDF / Word / HTML
↓
Parsing (extract structure)
↓
Chunking (split meaningful units)
↓
Metadata Enrichment
↓
Embedding
↓
Store in Vector DB

5. Online Pipeline (User Query)

User: "Công ty được làm gì?"
↓
Query Embedding
↓
Vector Search (Top-K)
↓
Re-ranking (optional but important)
↓
Build Context
↓
LLM Generate Answer

6. Hybrid Search (Production MUST HAVE)

Vector Search → hiểu nghĩa
Keyword Search → match chính xác
Final Score = 0.7 * vector + 0.3 * keyword
Best practice: Always combine BOTH

7. Re-Ranking Layer

Filter results again before LLM
Top 10 → Re-rank → Keep Top 3
Giúp giảm sai + tăng độ chính xác

8. Context Building

Combine chunks into prompt
Context: - Điều 5: Tự do kinh doanh - Điều 6: Nghĩa vụ doanh nghiệp Question: "Công ty được làm gì?"

9. LLM Layer

Input = Context + Question
↓
Output = Final Answer (grounded)

10. Metadata Storage

{ "text": "Tự do kinh doanh", "law": "Luật doanh nghiệp", "article": 5, "clause": 1 }
Metadata giúp: - Filter chính xác - Explain answer

11. Feedback Loop (Production Level)

User feedback
↓
Log queries + answers
↓
Improve retrieval / prompt
System becomes smarter over time

12. Monitoring

Latency
Accuracy
Search quality
If no monitoring → system will fail silently

13. Common Mistakes

Chunk too big → mất chính xác
No metadata → trả lời sai context
Only vector search → miss keyword
No re-ranking → noise cao

14. Simple Mental Model

Search Engine + Brain
Search Engine = Vector DB + Retrieval
Brain = LLM

15. One Sentence

Production RAG = Clean Data + Smart Search + Controlled LLM
