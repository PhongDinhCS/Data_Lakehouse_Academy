RAG End-to-End (Legal AI Training)

1. What is RAG?

RAG = Retrieval Augmented Generation
LLM does not guess → it uses your data
LLM + Search + Your Knowledge = Accurate Answer

2. Core Flow (Easy to Remember)

Documents → Chunk → Embedding → Vector DB
↓
User Query → Embedding → Similarity Search
↓
Top-K relevant chunks
↓
LLM (GPT / Llama)
↓
Final Answer

3. Full Pipeline Overview

Legal Documents (PDF, Word)
↓
Document Parsing (extract structure)
↓
Chunking (split by meaning)
↓
Embedding (text → vector)
↓
Vector Database (store meaning)
↓
Search + LLM → Final Answer

4. Example Input

Điều 5. Quyền của doanh nghiệp 1. Tự do kinh doanh 2. Lựa chọn ngành nghề

5. Document Parsing

Convert document into structured data
{ "article": 5, "title": "Quyền của doanh nghiệp", "clauses": [ {"clause": 1, "text": "Tự do kinh doanh"}, {"clause": 2, "text": "Lựa chọn ngành nghề"} ] }

6. Chunking

Split by meaning (not by size)
Chunk 1: "Tự do kinh doanh" Chunk 2: "Lựa chọn ngành nghề"

7. Metadata

Add context for accuracy
{ "text": "Tự do kinh doanh", "metadata": { "law": "Luật doanh nghiệp", "article": 5, "clause": 1 } }

8. Embedding

Convert text into vector (meaning)
"Tự do kinh doanh" → [0.21, -0.44, 0.88, ...]

9. Vector Database

Store vectors for semantic search
Similar meaning → similar vectors

10. Query Flow

User: "Công ty được làm gì?"
↓
Convert query → vector
↓
Search similar vectors
↓
Retrieve relevant chunks

11. LLM Generation

LLM receives context + question
Context: "Tự do kinh doanh" Question: "Công ty được làm gì?"
↓
Final Answer:
Doanh nghiệp có quyền tự do kinh doanh theo Điều 5.

12. Simple Code Example

from sentence_transformers import SentenceTransformer import faiss import numpy as np texts = ["Tự do kinh doanh", "Lựa chọn ngành nghề"] model = SentenceTransformer('all-MiniLM-L6-v2') embeddings = model.encode(texts) index = faiss.IndexFlatL2(embeddings.shape[1]) index.add(np.array(embeddings)) query = "Công ty được làm gì?" q_vec = model.encode([query]) D, I = index.search(q_vec, k=1) print(texts[I[0][0]])

13. System Flow Summary

User Query ↓ Embedding ↓ Vector DB Search ↓ Top-K Results ↓ LLM ↓ Final Answer

14. Key Takeaways

RAG = Search + LLM
Vector DB = understand meaning
Metadata = accuracy
Chunk by logic, not by length
